The toolbar gives your users formatting controls for document editing. Three paths, depending on framework and how much UI you want to own.

## Choose your approach

| | Built-in | Bring Your Own UI | Headless toolbar |
|---|---|---|---|
| Setup | `toolbar: '#toolbar'`. SuperDoc renders the UI | `useSuperDocCommand` from `superdoc/ui/react`. You render the buttons | `createHeadlessToolbar()`. Framework-agnostic substrate |
| Framework | Any | React | React, Vue, Svelte, vanilla JS, Angular |
| Re-render granularity | One toolbar redraw per command change | One re-render per button per command change | One snapshot per change; you pick how to apply it |
| Best for | Standard editing, quick prototypes | New React apps, design-system integration | Existing integrations, non-React apps |

**Recommendation.** New React apps should reach for [Bring Your Own UI](/bring-your-own-ui/toolbar-and-commands) first. The typed React surface (`useSuperDocCommand`, `ui.commands.register`) re-renders one button at a time and ships with companion hooks for selection, comments, and tracked changes. `createHeadlessToolbar()` stays the right pick for non-React apps and existing integrations; both paths read from the same internal substrate.

## Quick start

<Tabs>
  <Tab title="Built-in">
    Point `toolbar` at a container element. SuperDoc renders the UI for you.

    ```javascript
    new SuperDoc({
      selector: '#editor',
      document: 'contract.docx',
      toolbar: '#toolbar',
    });
    ```
  </Tab>

  <Tab title="Bring Your Own UI (React)">
    `useSuperDocCommand` returns the live state for a single command. Bind it to your button.

    ```tsx
    import { useSuperDocCommand, useSuperDocUI } from 'superdoc/ui/react';

    function BoldButton() {
      const ui = useSuperDocUI();
      const { active, disabled } = useSuperDocCommand('bold');
      return (
        <button
          data-active={active}
          disabled={disabled}
          onClick={() => ui?.commands.get('bold')?.execute()}
        >
          B
        </button>
      );
    }
    ```

    See [Bring Your Own UI › Toolbar and commands](/bring-your-own-ui/toolbar-and-commands) for the full pattern.
  </Tab>

  <Tab title="Headless">
    Framework-agnostic substrate. You get state snapshots and an `execute` method, wire them to any UI.

    ```javascript
    import { createHeadlessToolbar } from 'superdoc/headless-toolbar';

    const superdoc = new SuperDoc({
      selector: '#editor',
      document: 'contract.docx',
    });

    const toolbar = createHeadlessToolbar({
      superdoc,
      commands: ['bold', 'italic', 'underline', 'font-size', 'undo', 'redo'],
    });

    toolbar.subscribe(({ snapshot }) => {
      // Update your UI with snapshot.commands
    });

    toolbar.execute('bold');
    ```
  </Tab>
</Tabs>
